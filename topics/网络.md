### 网络

```bash
'''
你要访问的ip是提前知道的。你知道有两个主机
ping | ssh = 物理连通(光纤) + 逻辑连通(ip vlan mac)
ip:
    1. 同一区域：只需要数据链路层的以太网交换机，ip要求同一网段（如192.168.1.x/24），必须同网段 IP + 同 VLAN（否则逻辑不通），MAC地址网卡出厂自带（全球唯一），一般无需配置，自动携带，不用网关外传
    2. 不同区域：需要网关，ip可以不同，但是需要配置，MAC永远自动处理，你只需配置IP、子网掩码、网关、DNS

    
交换机：同一区域网控制如何发，通过mac地址识别，不牵扯ip.
路由器：跨网段数据传送，负责把死亡数据送到公网，管ip
网卡(唯一mac，负责发)：
    一个网卡对应一个网段，不插网卡，改的是内置网卡配置（一般用于正常访问网络百度且是DHCP设置），多一个网卡就多一个网段。
    1. 物理网卡：对应电脑自身出场自带
    2. 半虚拟网卡：蓝牙
    3. 虚拟网卡：转换usb
    隔离性：测试设备的网络（VLAN 69）和公司办公网完全隔离，不会互相干扰
    安全性：即使测试设备有病毒/广播风暴，不会影响到公司内网
    稳定性：直连测试设备不经过公司交换机，延迟更低，不受公司网络拥堵影响

    
mac: 不需要配置，网卡出厂时 MAC 地址已烧录在硬件里（如 00:1B:44:11:3A:B7），系统会自动读取。
gateway: 网关，负责非统一网段的通信
dns: 域名系统，也就是将人类熟悉的名字转换成ip
broadcast:广播地址是统一区域网，所有人可以收到，但是会丢弃。
netmask:用于判断是否是同一网段


同一局域网下工作机制：
场景：你在 192.168.1.10 Ping 同办公室的测试设备 192.168.1.20
┌─────────────────────────────────────────────────────────────────┐
│                         局域网 (LAN)                             │
│  网段: 192.168.1.0/24  |  网关: 192.168.1.1（此时闲置）           │
│                                                                 │
│   你的电脑              二层交换机              测试设备          │
│  ┌──────────┐        ┌──────────┐         ┌──────────┐         │
│  │ 应用层   │        │          │         │ 应用层   │         │
│  │ Ping命令 │        │          │         │ ICMP回复 │         │
│  ├──────────┤        │          │         ├──────────┤         │
│  │ 网络层   │        │          │         │ 网络层   │         │
│  │ 源IP:192.168.1.10│          │         │ 源IP:192.168.1.20 │ │
│  │ 目的IP:192.168.1.20         │         │ 目的IP:192.168.1.10│ │
│  ├──────────┤        │ 查MAC表  │         ├──────────┤         │
│  │ 数据链路 │        │ 端口1↔端口2│        │ 数据链路 │         │
│  │ 源MAC:AA│───────→│          │────────→│ 源MAC:BB │         │
│  │ 目的MAC:BB│       │  (学习:AA在端口1)  │ 目的MAC:AA│        │
│  ├──────────┤        │          │         ├──────────┤         │
│  │ 物理层   │        │          │         │ 物理层   │         │
│  │ 网卡     │        │ 交换芯片  │         │ 网卡     │         │
│  └──────────┘        └──────────┘         └──────────┘         │
│                                                                 │
│  关键动作：                                                      │
│  1. 电脑发现目的IP是同一网段（192.168.1.x），不经过网关            │
│  2. 发送ARP广播："谁是192.168.1.20？告诉我你的MAC！"              │
│  3. 交换机泛洪到所有端口（同VLAN）                                │
│  4. 测试设备回复MAC地址（BB），交换机记录"BB在端口2"              │
│  5. 后续数据包直接单播到端口2，不再广播                            │
└─────────────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────────────────────┐
│                               广域网通信                                  │
│                                                                          │
│   私网LAN                    边界路由器(NAT)              互联网          │
│  ┌──────────┐              ┌──────────────┐           ┌──────────┐      │
│  │ 你的电脑  │              │   路由器      │           │  百度服务器 │      │
│  │ 192.168.1.10            │  LAN口:192.168.1.1        │ 14.215.177.38     │
│  │ 网关:192.168.1.1        │  WAN口:公网IP 223.5.5.5   │                  │
│  └────┬─────┘              └──────┬───────┘           └────┬─────┘      │
│       │                           │                        │            │
│  1.发现目的IP是外网                │                        │            │
│    (不是192.168.x.x)               │                        │            │
│    →必须发给网关                   │                        │            │
│       │                           │                        │            │
│  2.数据包:                         │                        │            │
│    源IP:192.168.1.10               │                        │            │
│    目的IP:14.215.177.38            │                        │            │
│    源MAC:AA(你的网卡)               │                        │            │
│    目的MAC:AA:BB:CC(路由器LAN口)    │                        │            │
│       │                           │                        │            │
│       └────────→ 路由器LAN口收到    │                        │            │
│                   拆解发现是外网IP   │                        │            │
│                   →转给WAN口处理    │                        │            │
│                                  ↓                        │            │
│                            3.NAT转换:                      │            │
│                              源IP改为223.5.5.5(公网)       │            │
│                              记录映射: 192.168.1.10:xxxx   │            │
│                              ←→ 223.5.5.5:yyyy             │            │
│                                  │                        │            │
│                            4.路由选择:                      │            │
│                              查路由表→电信→百度           │            │
│                                  │                        │            │
│       ←←←←←←←←←←←←←←←←←←← 5.百度回复给223.5.5.5           │            │
│                                  │                        │            │
│                            6.NAT还原:                      │            │
│                              查表发现是192.168.1.10的      │            │
│       ←←←←←←←←←←←←←←←←←←← 源IP改回192.168.1.10转发给你    │            │
│                                                                          │
│  关键区别：                                                               │
│  - 局域网：MAC地址主导，交换机转发，IP仅用于识别是否同网段                    │
│  - 广域网：IP地址主导，路由器转发，MAC每跳都变（LAN口MAC→WAN口下一跳MAC）      │
│  - NAT：把私网IP"伪装"成公网IP，否则百度无法回复（私网IP不能在互联网路由）      │
└──────────────────────────────────────────────────────────────────────────┘


协议层图（数据封装与解封装）
发送方（你）                                              接收方（服务器）
┌──────────────────────────────────────────────────────────────────────────────┐
│  应用层 (Layer 7)          SSH/Ping/HTTP                ←→ 应用层          │
│       ↓ 数据                                               ↑ 数据          │
│  表示层 (Layer 6)          加密/编码                      ←→ 表示层          │
│       ↓ 数据段                                             ↑ 数据段        │
│  会话层 (Layer 5)          会话管理                       ←→ 会话层          │
├──────────────────────────────────────────────────────────────────────────────┤
│  传输层 (Layer 4)          TCP/UDP + 端口号               ←→ 传输层          │
│       ↓ 数据包 (Packet)      [SYN][ACK][数据]              ↑ 数据包          │
│  网络层 (Layer 3)          IP头 + 源/目的IP                ←→ 网络层          │
│       ↓ 数据帧 (Frame)                                     ↑ 数据帧        │
│       [源IP:192.168.1.10]                                  [源IP:192.168.1.10]│
│       [目的IP:192.168.1.20] 或 [目的IP:14.215.177.38]       [目的IP:...]     │
├──────────────────────────────────────────────────────────────────────────────┤
│  数据链路层 (Layer 2)      MAC头 + 源/目的MAC              ←→ 数据链路层      │
│       ↓ 比特流 (Bits)        [AA:BB:CC→DD:EE:FF]           ↑ 比特流          │
│  物理层 (Layer 1)          电信号/光信号/无线电            ←→ 物理层          │
│       ↓ 网线/光纤/空气                                     ↑ 网线/光纤      │
└──────────────────────────────────────────────────────────────────────────────┘

关键概念对应：
┌──────────────┬─────────────────────────────────────────┬────────────────────┐
│ 层级         │ 核心设备/协议                           │ 关键字段           │
├──────────────┼─────────────────────────────────────────┼────────────────────┤
│ 应用层(L7)   │ SSH(22), HTTP(80), FTP(21)              │ 域名、URL          │
│ 传输层(L4)   │ TCP(可靠), UDP(快)                      │ 端口号             │
│ 网络层(L3)   │ 路由器、IP协议、ICMP(Ping)              │ 源IP、目的IP       │
│ 数据链路层(L2)│ 交换机、MAC地址、以太网、VLAN          │ 源MAC、目的MAC、VLAN ID │
│ 物理层(L1)   │ 网卡、网线、光纤、集线器                 │ 电压、光脉冲、无线电波│
└──────────────┴─────────────────────────────────────────┴────────────────────┘


配置 checklist：
IP地址同网段，子网掩码定范围，网关指向路由器，DNS负责查域名
MAC地址不用管，VLAN看环境（办公室不用，工厂要配）

      |------------------------------------------>|

   [ESTABLISHED]                              [ESTABLISHED]
      |                                           |

四次挥手：
          主动关闭方 (Client)                被动关闭方 (Server)
           |                                      |
   [ESTABLISHED]                          [ESTABLISHED]
           |                                      |
           |  ① FIN=1, seq=u                    |
           |  "我没数据发了，我要关闭发送通道"        |
           | -----------------------------------> |
           |                                      |
  [FIN_WAIT_1]                            [CLOSE_WAIT]
           |                                      |

           | <----------------------------------- |
           |                                      |

           |                                      |

           | <----------------------------------- |
           |                                      |
           |                                      |
  [TIME_WAIT]                             [LAST_ACK]
           |  ④ ACK=1, ack=w+1                  |

           | -----------------------------------> |
           |                                      |
           |          [等待 2MSL 时间...]         |

           |                                      |
```
